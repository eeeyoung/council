# Project COUNCIL: Scientific Symposium Framework
## Technical Specification & Implementation Roadmap

### 1. Vision & Core Philosophy
**COUNCIL** (Collaborative Open-source Universal Consortium for Industrial & Logical Research) is a multi-agent "Brain Trust" built on **crewAI**. Unlike standard linear agent workflows, COUNCIL implements a "Plan-Research-Debate-Audit" cycle designed for high-rigor scientific brainstorming.

**Key Design Principles:**
*   **Modular Architecture:** Every component (Agents, Tools, Orchestrators) is decoupled. Prompts are managed in separate YAML/JSON files for easy maintenance.
*   **Built on crewAI:** Leverages crewAI's `Agent`, `Task`, and `Crew` classes, but extends them with custom asynchronous "Thinking" phases.
*   **Human-in-the-Loop (HITL):** A mandatory state-management layer allows for real-time interruptions and branching.

---

### 2. System Architecture (Modular Design)

The system is divided into five distinct modules to ensure future-proofing and ease of maintenance:

| Module | Purpose | Tech / Class |
| :--- | :--- | :--- |
| **A. Curator** | Decomposes query and spawns the expert panel. | `ModeratorAgent(Agent)` |
| **B. Research Lab** | Parallel, asynchronous data gathering and tool use. | `AsyncProcess` + `ResearchTools` |
| **C. The Floor** | Sequential, context-aware debate. | `Crew(process=Process.sequential)` |
| **D. The Audit** | Double-host judgment loop (Synthesis vs. Critique). | `HostAAgent` & `HostBAgent` |
| **E. The Dossier** | Compiles evidence-backed final reports. | `RecorderAgent(Agent)` |

---

### 3. Phase-by-Phase Execution Logic

#### Phase A: The Panel Curation
*   **Moderator** analyzes the user query.
*   Dynamically generates **$N$ Expert Personas** (up to 8).
*   **GUI Step:** User approves/edits the panel.
*   *Implementation:* Uses a factory pattern to instantiate `crewai.Agent` objects with dynamic system prompts.

#### Phase B: Parallel Research (The "Thinking" Phase)
*   **Isolation:** Agents are forbidden from talking to each other yet.
*   **Action:** Each agent runs a `Task` with `async_execution=True`.
*   **Tools:** Access to `TavilySearchResults` and `PythonREPL`.
*   **Shared Library:** Findings (URLs/Snippets) are pushed to a `ChromaDB` vector store shared across the crew.

#### Phase C: The Sequential Symposium
*   **Floor Order:** Agents speak A → B → C.
*   **Context Injection:** Each agent's prompt is updated in real-time with the `transcript` of the previous speakers.
*   **Evidence Scorecard:** A background process (via the `RecorderAgent`) monitors the chat to map claims to citations in the Shared Library.

#### Phase D: The Audit Loop (Host A & B)
*   **Host A (Synthesizer):** Creates a unified hypothesis from the debate.
*   **Host B (Auditor):** A "Hostile Peer Reviewer" agent that checks for lazy consensus or ignored physical constraints.
*   **Iteration:** If Host B rejects, the Crew is re-initialized for a new round with a "Conflict Mandate."

---

### 4. The "Director's Console" (Interruption & Branching)

To allow the user to pause and intervene, the `crew.kickoff()` must be wrapped in a **Stateful Controller**:

1.  **Checkpointing:** After every `Task` completion, the current `CouncilState` is saved to a `SQLite` database.
2.  **Interrupt Hook:** A global `stop_event` flag is monitored. When triggered via the GUI, the loop pauses.
3.  **Branching:** If the user corrects an agent, a new "Thread" is created in the database. The user can "Rewind" the symposium to any previous agent's speech and resume with new parameters.

### 5. Implementation Roadmap for Coding AI

**Task 1: Basic Skeleton**
- Initialize a `crewai` project.
- Create a `CouncilState` class using `Pydantic` to track the transcript and evidence scorecard.
- Implement a `Moderator` agent that can output a JSON list of expert definitions.

**Task 2: Asynchronous Research**
- Build custom `crewai.Tools` for internet search and PDF parsing.
- Configure the initial "Thinking" tasks to run in parallel using Python's `asyncio`.

**Task 3: The Audit & Interrupt Logic**
- Develop the `Host B` system prompt (Peer Reviewer logic).
- Implement a `pause_resume` wrapper around the crew execution loop.

**Task 4: The Final Dossier**
- Create a `Recorder` agent that formats the output into a professional Markdown research paper with a full bibliography.
